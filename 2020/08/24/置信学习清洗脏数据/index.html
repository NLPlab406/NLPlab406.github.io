    置信学习

    每次我们跑模型时是不是经常都是不管数据，或者直接分析拿到手的数据，而从来没担心过数据标注是否正确？

    即使我们知道数据标注有错误，那我们总不能一个一个检查标签是否正确吧？

    即使数据量很小，数据标注往往是涉及到专业的领域的，那是不是我们也不知道标注是否正确？

    • 显然，如果错误标签的数据（以下称为噪声标签）参与训练，卧～，本来效果就不好，你还让模型拿错误数据学习，是不是很奇葩～～

    下面我们一步步提出解决方案（置信学习）

    训练数据标记为错误标签的两种情况

    • 测试数据badcase本来错的，预测为正确标签————————>错误
    • 测试数据badcase本来错的，预测为错误标签————————>正确

    置信学习（CL）

    • 原理：CL 基于噪声数据剪枝的原理（与修复标签错误或修改损失函数相反），以计数的方式对噪声进行评估，并对示例进行排序以进行置信训练（而不是通过精确的概率加权）。

    优势

    • 最大的优势：可以用于发现标注错误的样本！
    • 不需要超参数，只需使用交叉验证来获得样本外的预测概率。

    • 与模型无关，可以使用任意模型，不像众多带噪学习与模型和训练过程强耦合。

    • 置信学习找出的「标注错误的样本」，不一定是真实错误的样本，这是一种基于不确定估计的选择方法。

    • 交叉验证，找到可能是错误的标签数据
    python
    • 无需交叉验证，直接剔除错误标签
    python
    • 输入 s:噪声标签 psx: n x m 的预测概率，通过交叉验证获得
    Code

    置信学习的三个步骤

    • Count：估计噪声标签和真实标签的联合分布；
    • Clean：找出并过滤掉错误样本；
    • Re-Training：过滤错误样本后，重新调整样本类别权重，重新训练；

    应用领域

    • 描述噪声标签
    • 寻找标签误差
    • 采用噪声标签学习
    • 寻找本体论问题
