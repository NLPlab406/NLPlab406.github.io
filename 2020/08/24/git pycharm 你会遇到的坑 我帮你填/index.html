写在文章开头

  • 本文的目的其实主要是介绍pycharm使用github对“傻瓜式多人分布式编程”的问题总结，也许其中有你遇到过的……

  • 下面总结了十个问题，多数是在使用pycharm经常会遇到的问题

常见问题总结

1.当你在code中完成一个当前的功能A（并没有写完）时，但此时有一个新增优先级更高的任务B（与A完全不相关）需要马上进行，git的操作是怎么样的？

答：第一步，将master最新code合并到当前分支A

第二步，先commit保存并提交当前A分支代码（图1，图2，只需要commit），点击commit

第三步，切换到master，pull主分支code

第四步，在master上checkout新分支B

2.在第1问的基础之上，当前是在B（未完成）所在分支，但是你在A的分支上在程序中执行了一个小的改动，那么怎么把A的pr（pull request）发出来？

答：第一步，先commit保存并提交当前B分支代码（图1，图2，只需要commit），点击commit。

第二步，提交修改后的A分支，打开github

第三步，新建pr

3.假设a,b两人在协同完成同一个功能，都是在github的同一个远程分支branch-A上，当b做了改动，当然a也可能在同时做改动，此时git操作应该怎么做？

第一步，a将brach-A最新code合并到当前自己的分支brach-A-function1

第二步，先commit保存并提交当前a的本地brach-A-function1分支代码（图1，图2，只需要commit），点击commit&push。

第三步， b需要在brach-A需要执行git pull，pull最新code，并新建分支brach-A-function2。

第四步，如果需要提交code，需要和a执行同样的操作。

4.git commit和git push命令有什么区别？

答：git commit操作的是本地库，git push操作的是远程库。

​ git commit是将本地修改过的文件提交到本地库中。

​ git push是将本地库中的最新信息发送给远程库。

​ 那有人就会问，为什么要分本地commit和服务器的push呢？

​ 因为如果本地不commit的话，修改的纪录可能会丢失。
​ 而有些修改当前是不需要同步至服务器的，所以什么时候需要同步再push到服务器由用户自行选择。

5.当git pull远程分支代码时，产生冲突怎么办？

答：需要merge你的本地代码和远程仓库的code。

6.Github在本地各个分支的命名规则是什么，怎么命名更加合理？

答：根据功能命名，首先符合功能，其次简单易懂，切忌使用自己的名称命名分支。

7.当你已经完成一个功能A，其所在分支为brach-A，且该分支已经合并到了远程分支上。在之后的工作中，你的新功能可以写到之前的branch-A中吗？如果可以，请说明问什么不推荐这样做，推荐做法又是什么？

答：可以，但是不推荐，因为会有很多历史记录存在，推荐删除该功能的本地分支，再创建新的分支，完成新的功能。

8.git stash命令在什么时候会用到，为什么我们不推荐使用该命令？

答： git add会把本地code提交到git track list，

​ git stash是将在当前分支是那个修改的记录缓存起来，放在缓存区里面

​ 但是git stash的作用范围是整个工作目录，不是某个分支，缓冲区原本就是个共享栈，很多分支都在用，pop的时候只会pop存储在最顶端的code，但是 这个很有可能是别人stash的。

9.假如你有一些代码不想提交到远程怎么办 ？

答：首先在.gitignore中加入该文件或者目录

之后提交并push该文件

10.git 缓冲区是什么，位置在本地还是远程？
答：本地
